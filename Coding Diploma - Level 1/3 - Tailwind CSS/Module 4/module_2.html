2.1 The Basics

A Utility first approach is instead of using individual classes for each element, we use a utility class. This is a class named in our markup which allows us to control our style, while being quick and efficient.

2.2 Core Utility Components

Tailwind uses a shorthand syntax as opposed to Regular CSS. Making it easier to change and create styling. 

e.g., 
    Regular CSS: .container { display: flexbox; background-color: #2980B9; padding: 2px; margin: 2px;}
    Tailwind CSS: <div> class='flex bg-blue-500 p-2 m2'<div>

2.3 Responsive Design Principles

Tailwind has desing utilities that make responsive design a lot more attainable and intuitive than CSS.

We can use elements like sm, md, lg, xl and 2-10xl to denote which screen type we will be using. Allowing consistent and flexible designs.

640px -> sm

768px -> md

1024px -> lg

1280px -> xl

e.g., <div>class='text-xl text-blue-400 sm:text-red-500 md:text-green-500'</div>
