A Deeper Look at Variables

In JavaScript, variables are used to store data values. They can be declared using three keywords: var, let, and const. Each has its own scope and use cases.

Variables can hold different types of data, including numbers, strings, booleans, objects, and arrays or a mix of all of them. Understanding how to declare and use variables is fundamental to programming in JavaScript. 


var name = "John Doe"; // Variable declaration using var
let age = 30; // Variable declaration using let
const PI = 3.14159; // Constant declaration using const

Using var to Declare a Variable

The var keyword is used to declare a variable that can be re-assigned later. It has function scope, meaning it is accessible within the function it is declared in, or globally if declared outside of any function. However, using var is generally discouraged in modern JavaScript due to its function scope, which can lead to unexpected behavior. Instead, it is recommended to use let or const.


When a variable is declared with var outside of a function, it becomes a globally scoped variable. 


var globalVar = "I am a global variable"; // Global variable
function myFunction() {
  console.log(globalVar); // Accessible here
}
myFunction(); // Outputs: I am a global variable
console.log(globalVar); // Outputs: I am a global variable

When a variable is declared with var inside a block (like an if statement), it is still accessible outside that block due to its function scope. 


var blockVar = "I am a block variable"; // Block variable
if (true) {
  var blockVar = "I am still accessible"; // Re-declaring the same variable
  console.log(blockVar); // Outputs: I am still accessible
}
console.log(blockVar); // Outputs: I am still accessible

When a variable is declared with var inside a function, it is only accessible within that function. 


function myFunction() {
  var localVar = "I am a local variable"; // Local variable
  console.log(localVar); // Accessible here
}
myFunction();
console.log(localVar); // Error: localVar is not defined

Using let to Declare a Variable

The let keyword is used to declare a block-scoped variable. This means that the variable is only accessible within the block it is declared in, such as within an if statement or a loop. This helps prevent issues with variable scope and makes code more predictable. 


let blockScopedVar = "I am a block-scoped variable"; // Block-scoped variable
if (true) {
  let blockScopedVar = "I am only accessible here"; // New variable in this block
  console.log(blockScopedVar); // Outputs: I am only accessible here
}
console.log(blockScopedVar); // Outputs: I am a block-scoped variable

When a variable is declared with let inside a block, it is not accessible outside that block, preventing potential conflicts with other variables. 


let anotherBlockVar = "I am another block variable"; // Another block variable
if (true) {
  let another
BlockVar = "I am only accessible in this block"; // New variable in this block
  console.log(anotherBlockVar); // Outputs: I am only accessible in this block
}
console.log(anotherBlockVar); // Outputs: I am another block variable

When a variable is declared with let inside a loop, it is only accessible within that loop. 


for (let i = 0; i < 5; i++) {
  let loopVar = "I am a loop variable " + i; // Loop variable
  console.log(loopVar); // Outputs: I am a loop variable 0, 1, 2, 3, 4
}
console.log(loopVar); // Error: loopVar is not defined

The let keyword is preferred over var for declaring variables in modern JavaScript due to its block scope, which helps avoid unintended variable conflicts.

Using const to Declare a Variable

The const keyword is used to declare a constant variable, which means its value cannot be re-assigned after it has been set. Like let, it is block-scoped, meaning it is only accessible within the block it is declared in. 


const constantVar = "I am a constant variable"; // Constant variable
if (true) {
  const constantVar = "I am only accessible here"; // New constant in this block
  console.log(constantVar); // Outputs: I am only accessible here
}
console.log(constantVar); // Outputs: I am a constant variable

When a variable is declared with const inside a block, it is not accessible outside that block, similar to let. 


const anotherConstantVar = "I am another constant variable"; // Another constant variable
if (true) {
  const anotherConstantVar = "I am only accessible in this block"; // New constant in this block
  console.log(anotherConstantVar); // Outputs: I am only accessible in this block
}
console.log(anotherConstantVar); // Outputs: I am another constant variable

When a variable is declared with const inside a loop, it is only accessible within that loop, similar to let. 


for (let i = 0; i < 5; i++) {
  const loopConst = "I am a loop constant " + i; // Loop constant
  console.log(loopConst); // Outputs: I am a loop constant 0, 1, 2, 3, 4
}
console.log(loopConst); // Error: loopConst is not defined

The const keyword is used for variables that should not change after they are set, providing a way to create immutable values in JavaScript.


If the const is an object or an array, you can still modify, add or remove items. 


const person = {
  name: "John Doe",
  age: 30
};

person.age = 31; // Modifying a property of the object
person.city = "New York"; // Adding a new property to the object
delete person.name; // Deleting a property from the object

console.log(person); // Outputs: { age: 31, city: "New York" }

Variables & Data Types

JavaScript is a dynamically typed language, meaning it understands the data type being used without needing us to declare the type being used.

Variables can hold different types of data, including numbers, strings, booleans, objects, and arrays. The type of data a variable holds can change during the execution of the program. 


// String: Represents a sequence of characters enclosed in single or double quotes.
let name = "John Doe"; 
let message = 'Hello, World!';

// Number: Represents numeric values, both integers and floating-point numbers.
let age = 30;
let pi = 3.14159;

// Boolean: Represents a logical value, either true or false.
let isActive = true;
let isComplete = false;

// Undefined: Represents a variable that has been declared but not assigned a value.
let notAssigned;
let emptyValue;

// Null: Represents an intentional absence of any value.
let emptyObject = null;

// Object: Represents a collection of key-value pairs, where keys are strings and values can be any data type. Objects can containe properties and methods that are used to store and manipulate more complex data structures.
let person = {
  name: "John Doe",
  age: 30,
  isActive: true
};

// Array: Represents an ordered list of values, which can be of any data type. Arrays are used to store multiple values in a single variable.
let numbers = [1, 2, 3, 4, 5];
let mixedArray = ["Hello", 42, true, null, { key: "value" }];

// Function: Represents a block of reuseable code designed to perform a specific task. Functions can take parameters and return values. They are used to encapsulate and organise code.
let greet = function(name) {
  return "Hello, " + name + "!";
};

Different Types of Operators

JavaScript provides various types of operators to perform operations on variables and values. These include arithmetic, assignment, comparison, logical, and bitwise operators.


Arithmetic Operators

Used to perform mathematical operations like addition, subtraction, multiplication, division, and modulus. 

let a = 10;
let b = 5;
let sum = a + b;
let subtraction = a - b;
let multiplication = a * b;
let division = a / b;
let modulus = a % b; // Remainder of division

Assignment Operators

Used to assign values to variables. The basic assignment operator is =, but there are also compound assignment operators like +=, -=, *=, and /=.


Equality comparisons and Sameness

JavaScript provides several operators to compare values. The equality operator == checks for value equality, while the strict equality operator === checks for both value and type equality. Similarly, the inequality operator != checks for value inequality, and the strict inequality operator !== checks for both value and type inequality. 

let x = 10;
let y = "10";

console.log(x == y); // true (value equality)
console.log(x === y); // false (strict equality, different types)
console.log(x != y); // false (value inequality)
console.log(x !== y); // true (strict inequality, different types)

Comparison Operators

Used to compare values and return a boolean result. These include greater than (>), less than (<), greater than or equal to (>=), and less than or equal to (<=). 

let a = 10;
let b = 5;
console.log(a > b); // true (10 is greater than 5)
console.log(a < b); // false (10 is not less than 5)
console.log(a >= b); // true (10 is greater than or equal to 5)
console.log(a <= b); // false (10 is not less than or equal to 5)

Logical Operators

Used to combine multiple boolean expressions. The logical AND operator (&&) returns true if both operands are true, the logical OR operator (||) returns true if at least one operand is true, and the logical NOT operator (!) negates a boolean value. 

let a = true;
let b = false;
console.log(a && b); // false (both must be true)
console.log(a || b); // true (at least one is true)
console.log(!a); // false (negates the value)

Bitwise Operators

Used to perform operations on binary representations of numbers. These include bitwise AND (&), bitwise OR (|), bitwise XOR (^), left shift (<<), right shift (>>), and unsigned right shift (>>>). 

let a = 5; // Binary: 0101
let b = 3; // Binary: 0011
console.log(a & b); // 1 (Binary: 0001, bitwise AND)
console.log(a | b); // 7 (Binary: 0111, bitwise OR)
